紧贴底部的页脚

发表于 2017-05-01

在网页设计中，存在一个相当古老但又常见的问题，它是前端工程师绕不开的坎。这个问题可以简单的概括如下：有一个具有块级样式的页脚（比如它设置了背景或阴影），当页面内容足够长时它一切正常，而当页面较短时就会出现问题。此时的问题在于，页脚不能像我们期望中的那样”紧贴”在视口的最底部，而是紧跟在内容得到下方。

这个问题不仅普遍存在，而且乍看起来确实相当简单。但是CSS2.1基本上拿它没办法；几乎所有经典解决方案都需要给页脚设置固定高度，而这显然是不健壮的，也是不实际的。

固定高度的解决方案

下面是示例的html代码：

<header>
    <h1>Site name</h1>
</header>
<main>
    <p>Bacon Ipsum dolor sit amet…
    <!-- Filler text --></p>
</main>
<footer>
    <p>© 2017 No rights reserved.</p>
    <p>Made with ♥ by jose.</p>
</footer>

假设这个页脚的文本永远不可能折行，那我们就可以推算出它实际所占的高度：

2行 * 行高 + 3 * 段落的垂直外边距（margin） + 页脚的垂直内边距

同样，我们也可以得出页头的高度，接下来，借助视口相关的长度单位以及calc()函数，只需一行CSS代码就可以表达出这种尺寸关系，从而把页脚”固定”到底部：

main {
    min-height: calc(100vh - 页脚高度 - 页头高度);
    /* 避免内边距或边框搞乱样式的计算*/
    box-sizing: border-box;
}

这个方法是有效的，而且它似乎比那些需要固定高度的方案还要稍微好一些，主要好在代码极其精简。不过，如果页面布局不是这么简单的话，那这个方法就完全不适用了。它不仅要求我们确保页脚内的文本永不换行，而且每当我们改变页脚的尺寸时，都需要手动调整min-height的值。想必在这个美好的新时代，我们还有更好的办法，对吗？

更灵活的解决方案

Flexbox对于此类问题同样是完美的选择，只寥寥几行CSS代码就可以完美达成十分灵活的布局效果，而且不需要我们负责的计算或者是添加多余的html元素。

  • 首先我们需要对body元素设置display: flex，因为它是这三个主要区块的父元素，当父元素获得这个属性之后，就可以对其子元素出发”伸缩盒布局模型”。我们还需要把flex-flow设置为column，否则子元素会被水平排放在一行上：
body {
   dispaly: flex;
   flex-flow: column;
}

此时，页面看起来与没有启用Flexbox的情况似乎是一样的，因为所有元素都占据了整个视口的宽度，而他们的高度也都是由其自身的内容决定的。也就是说，我们还没有真正发挥Flexbox的威力。

为了完全释放它的魔力，我们首先要把<body>的min-height属性指定为100vh，这样它就至少会占据整个视口的高度。不过，现在整个页面的表现跟之前并无区别，原因在于，虽然我们已经为整个body元素指定了最小高度，但各个子元素的高度仍然是以各自的内容为准（按照CSS规范的说法，它们的高度仍然由内部因素决定）。

此时，我们所期望的是，页头和页脚的高度由其内部因素决定，而内容区块的高度应该可以自动伸展并占满所有可用的空间。我们只要给<main>这个容器的flex属性指定一个大于0的只，就可以实现这个效果了：

body {
    display: flex;
    flex-flow: column;
    min-height: 100vh;
}
main { flex: 1; }

这样就可以了！我们只需要四行简单的代码，就完美实现了紧贴底部的页脚。Flexbox是不是相当霸气？
